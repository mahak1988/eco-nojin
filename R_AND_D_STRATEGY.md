## ۱. خلاصه اجرایی

این سند نقشه‌ راه توسعه پلتفرم اکونوجین را به یک پلتفرم **بین‌المللی بی‌نظیر** در حوزه پایش و شبیه‌سازی محیط‌زیست ارائه می‌دهد. هدف:

- **۱۰ زبان زنده دنیا** (پوشش ۵+ میلیارد گویشور)
- **۸ شبیه‌ساز علمی** (اقلیم، هیدرولوژی، زراعت، کربن، فرسایش خاک، سیلاب، خشکسالی، تنوع زیستی)
- **۱۵+ منبع داده ماهواره‌ای رایگان** (Sentinel, Landsat, MODIS, GPM, SMAP, GRACE, ICESat-2, GEDI)
- **۱۰ سناریوی آماده** برای مدیران، کارشناسان، دانشجویان و کشاورزان
- **معماری چندمخاطبی** با رابط‌های تخصصی برای هر گروه کاربری

## ۲. بنچمارک بین‌المللی

### ۲.۱ پلتفرم‌های مورد بررسی

| پلتفرم | نقطه قوت متمایز |
|--------|-----------------|
| Google Earth Engine | پردازش سمت سرور در مقیاس سیاره‌ای |
| Sentinel Hub | بصری‌سازی نواری سفارشی فوری |
| NASA Worldview | مرور بدون لاگین با ~۱۰۰۰ لایه |
| Copernicus CDS | ERA5 + CMIP6 (داده اقلیمی ۱۹۴۰ تاکنون) |
| FAO GeoNetwork | ۵۰۰۰+ دیتاست کشاورزی |
| UNEP WESR | داشبورد فدراتیو چندموضوعی |
| NOAA Climate Explorer | پیش‌بینی اقلیمی سطح شهرستان |
| Global Forest Watch | هشدارهای نزدیک به بلادرنگ |
| OpenWeatherMap | API ساده + endpoint کشاورزی |
| SoilGrids (ISRIC) | نقشه خاک جهانی با ML |

### ۲.۲ یافته‌های کلیدی

۱. **هیچ پلتفرمی همه‌جانبه نیست** — هیچ رقیبی همه ۸ نوع شبیه‌ساز را در یک جا ندارد. این فرصت متمایزسازی ماست.

۲. **الگوی "Subscribe-and-Alert"** (مشابه Global Forest Watch) بیشترین تأثیر کاربری را دارد — هشدار ایمیلی/پیامکی وقتی شرایط بحرانی رخ می‌دهد.

۳. **مدل پردازش سمت سرور** (GEE pattern) ضروری است — کاربر نباید داده‌های چند گیگابایتی دانلود کند.

۴. **STAC (SpatioTemporal Asset Catalog)** استاندارد فدراسیون داده است — باید پذیرفته شود.

۵. **PWA + آفلاین** برای جمع‌آوری داده میدانی حیاتی است (مناطق روستایی بدون اینترنت).

### ۴.۳ طراحی موتور شبیه‌ساز

هر موتور اینترفیس یکسان دارد:

```typescript
interface SimulatorEngine<P, R> {
  id: string;
  nameKey: string;          // i18n key
  descriptionKey: string;
  icon: string;
  audience: SimulatorAudience[];  // ["farmer", "expert", "student", "manager"]
  parameters: SimParameterDef<P>[];
  presets: SimPreset<P>[];
  run: (params: P) => Promise<SimResult<R>>;
  visualize: (result: R) => VisualizationSpec;
}
```

## ۷. ویژگی‌های چندمخاطبی

### ۷.۱ نقش‌ها و رابط‌های اختصاصی

| مخاطب | رابط اختصاصی | ویژگی‌های کلیدی |
|-------|-------------|----------------|
| **کشاورز** | `/farmer` | تقویم کشاورزی، وضعیت محصول، هشدار آفت، قیمت بازار، پیش‌بینی آب‌وهوا |
| **دانشجو** | `/student` | آزمایشگاه مجازی، تمرین شبیه‌ساز، گام‌به‌گام آموزشی، آزمون |

### ۷.۲ داشبورد تطبیقی

هر کاربر بر اساس نقش خود داشبورد متفاوتی می‌بیند:

- کشاورز → وضعیت مزرعه + آب‌وهوا + قیمت
- دانشجو → آزمایشگاه + دوره‌ها + پیشرفت
- کارشناس → تحلیل‌ها + گزارش‌های فعال + ابزارها
- مدیر → KPIها + هشدارها + تصمیم‌یار
- پژوهشگر → داده‌ها + نمودارها + استنادها
