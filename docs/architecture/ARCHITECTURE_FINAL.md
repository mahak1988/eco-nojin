# معماری نهایی اکو نوژین - نسخه 1.0

**تاریخ:** 2026-06-15
**نسخه:** 1.0.0

## نمای کلی معماری

پلتفرم اکو نوژین از معماری **Domain-Driven Design (DDD)** با **Coupling Engine مرکزی** استفاده می‌کند که تمام سرویس‌ها را به‌صورت یکپارچه مدیریت می‌نماید.

## لایه‌های معماری

### 1. لایه ارائه (Presentation Layer)
- **داشبوردهای تخصصی:** مدیر حوضه، کشاورز، سرمایه‌گذار، سیاست‌گذار
- **API Gateway:** نقطه ورود یکپارچه برای تمام درخواست‌ها
- **WebSocket:** ارتباط بلادرنگ با دستگاه‌های IoT

### 2. لایه Orchestration
- **Coupling Engine:** اتصال تمام سرویس‌ها و مدیریت گردش کارها
- **Orchestration Service:** مدیریت گردش کارهای پیچیده بین دامنه‌ها
- **Workflow Manager:** اجرای و پایش گردش کارها

### 3. لایه دامنه (Domain Layer)

#### 3.1 دامنه‌های اصلی
- **drought:** مدیریت خشکسالی و پایش شاخص‌های آبی
- **soil_water:** مدیریت خاک و آب، محاسبات RUSLE
- **financial:** مدیریت مالی، NPV، IRR، اعتبارات کربن
- **iot:** مدیریت دستگاه‌های IoT و داده‌های بلادرنگ
- **hydrology:** مدل‌سازی SWAT و WEAP
- **remote_sensing:** سنجش‌ازدور و شاخص‌های طیفی
- **mrv:** پایش، گزارش‌دهی و راستی‌آزمایی IPCC
- **dashboard:** داشبوردهای مدیریتی
- **training:** آموزش و ظرفیت‌سازی
- **psychology:** ارزیابی‌های روانشناختی

#### 3.2 ساختار هر دامنه
هر دامنه از ساختار استاندارد DDD پیروی می‌کند:
- **models:** مدل‌های داده‌ای
- **schemas:** اسکیماهای Pydantic
- **repositories:** عملیات پایگاه داده
- **services:** منطق تجاری
- **routers:** API endpoints

### 4. لایه زیرساخت (Infrastructure Layer)
- **PostgreSQL:** پایگاه داده اصلی
- **Redis:** کش و صف پیام
- **Mosquitto:** MQTT broker برای IoT
- **Blockchain:** قراردادهای هوشمند (SeedToken, GaiaCertificate)
- **Prometheus/Grafana:** مانیتورینگ

### 5. لایه یکپارچه‌سازی خارجی
- **سنجش‌ازدور:** Sentinel-2, Landsat, MODIS
- **مدل‌های هیدرولوژیک:** SWAT, WEAP, AquaCrop
- **بلاکچین:** Ethereum/Polygon
- **IoT:** LoRaWAN, MQTT
